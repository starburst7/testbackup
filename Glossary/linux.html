Gaining Temporary Admin Privileges

If you install ubuntu and you do not have a root account setup. Chances are you have a sudu account setup. Alot of manuals use "sudo s" but remaining active means you can get hacked. Thus, for ubuntu it is better to use ""sudo -s" which gives temporary access on a TIMER(which linux decides). It is useful as sometimes need to perform many restricted tasks but need to avoid being asked our password each time. TYPE EXIT to manually disable temporary admin

Changing PATH for Scripts

NOTE 1) the image shows paths in order that the system searches
2)when entering command to execute a script (i.e ./ ScritName ), the "./" You want to create your own personal bin folder(Linux will automatically remember it. You don't need to do anything). This removes the need to enter "./" in terminal. MORE IMPORTANTLY, you do not want to execute scripts from your CURRENT dir(note / denotes current dir) as hackers can slip malware in the current directory which could have the same name as a regular command. Keep in mind that placing this script in this bin will ONLY be usable by you. 3) to create script that is executable for other users on the computer, then place this in /usr/local/bin
