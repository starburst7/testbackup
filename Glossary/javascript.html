Using Strict Mode to prevent accidental Global variables

Insert at top of JS file. purpose : throw an error if you accidentally forget to add "var" identifier(thus making a variable global by accident) Avoid naming conflicts for large programs.
Prevents the declaration of a variable without using var keyword basically or else JavaScript engine may create a new global variable if it were non-existent before.
Example:

var calculateTax = function( subtotal, taxRate) {
tax= subtotal* taxRate; // no var keyword so tax is treated as global
tax= tax.toFixed(2); } ;

Best Coding practice for working with local and global variables

1. Use local variables whenever possible
2. always use strict mode put "use strict"; at top of code
3. declare the variables that are used by a function at the start of function before you use them.
4. If there are numerous functions, each function should only calls functions that precede it in the code(logical ordering)
